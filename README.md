# Бот клуба Win-Win

Telegram-бот: запись на падел-тренировки и турниры, заявки на Сайкл и Camp в Турцию, правила оплаты.

## Что умеет
- 🎾 Запись на падел-тренировку (4 формата с ценами) → показывает реквизиты для оплаты
- 🏆 Запись на турнир (5000 ₽) → реквизиты
- 🚴 Сайкл — собирает имя + контакт, отправляет заявку администратору
- 🌴 Camp в Турцию — собирает имя + контакт, отправляет заявку администратору
- 📋 Правила оплаты и отмены (возврат за 2 дня)

Заявки на Сайкл и Турцию приходят администратору (@Olga_1Pervaya) в Telegram.
VK и Макс пока не подключены.

## Установка и запуск

1. Установить Python 3.10+
2. Установить зависимости:
   ```
   pip install -r requirements.txt
   ```
3. Скопировать `.env.example` в `.env`:
   ```
   copy .env.example .env
   ```
4. Узнать свой ADMIN_ID: запустить бота (шаг 5), написать ему `/id`, скопировать число в `.env`, перезапустить.
5. Запустить:
   ```
   python bot.py
   ```

Локально бот работает через **polling** (опрашивает Telegram сам) — ничего настраивать не надо.

## Деплой на Render (бесплатно)

Бот сам определяет режим:
- если есть публичный адрес (`RENDER_EXTERNAL_URL` на Render) → **webhook**;
- если адреса нет (локальный запуск) → **polling**.

Шаги:
1. Залить код в Git-репозиторий (GitHub/GitLab).
2. На Render создать **Web Service** из этого репозитория (Docker определится автоматически).
3. В разделе **Environment** задать переменные:
   - `BOT_TOKEN` — токен от @BotFather
   - `ADMIN_ID` — числовой ID администратора
   - `PROXY_URL` — **оставить пустым** (с сервера Render Telegram доступен напрямую)
   - `WEBHOOK_URL` — **не задавать**, Render подставит адрес сам
4. Нажать **Deploy**. После сборки бот сам зарегистрирует webhook.

Обновление кода: правишь локально → `git push` → Render автоматически пересобирает и передеплоивает.
